# リアルタイム回答生成アシスタント 企画提案書

「会話の最中に、AIが聞き取った内容から回答候補をリアルタイムに画面表示するソフト」の企画・技術提案。
想定プラットフォーム: **Windows PC / iPhone**

---

## 1. コンセプト

会話(対面・通話・Web会議・配信)の音声をリアルタイムに文字起こしし、
LLM(Claude API など)が「次にあなたが言うべきこと」の候補を数秒以内に画面へ表示する。

```
相手の発話 → 音声キャプチャ → ストリーミング文字起こし(STT)
          → LLMへ逐次投入 → 回答候補を2〜3件生成 → 画面にカンペ表示
```

目標レイテンシ: 相手が話し終わってから **1〜2秒以内** に最初の回答候補が出ること。

---

## 2. できること(ユースケース提案)

### A. 会議・商談コパイロット(Windows向け・本命)
- Zoom / Teams / Meet の音声をPC内部でキャプチャして全発言を文字起こし
- 質問が飛んできたら、事前に読み込ませた資料(製品情報・過去議事録)を根拠に回答案を表示
- 「言い回し3パターン」(丁寧/簡潔/データ根拠つき)を同時提示
- 会議終了後に議事録・TODO・決定事項を自動生成(副産物として強力)

### B. 面接・試験対策モード
- 想定問答集+職務経歴書を事前登録
- 面接官の質問を聞き取り、自分の経歴に沿った回答骨子を箇条書きでカンペ表示
- 練習モード: AIが面接官役になり、回答へのフィードバックも返す

### C. 営業電話・カスタマーサポート支援
- 通話中に FAQ・料金表・在庫情報から即座に回答案を提示
- 相手の温度感(不満・興味あり)を判定してトークスクリプトを切り替え
- ※通話録音は相手の同意が必要(後述の注意点参照)

### D. ライブ配信アシスタント(既存のYouTube企画と親和性が高い)
- 配信中のコメントと自分の発話を拾い、返しのネタ・次のトークテーマを提案
- 沈黙が続いたら話題を自動サジェスト
- 本リポジトリのVOICEVOX自動化資産と組み合わせれば「AI共演者」化も可能

### E. 語学・翻訳サポート
- 相手の外国語発話を翻訳表示+自然な返答例を母語と外国語で併記
- 対面会話ならiPhone単体で完結できる

## 3. プラットフォーム別の実現性

| 項目 | Windows PC | iPhone |
|---|---|---|
| マイク音声の取得 | ○ 容易 | ○ 容易 |
| 相手側音声(通話/会議アプリ)の取得 | ◎ WASAPIループバックでPC再生音を丸ごと取れる | ✕ iOSの制限で他アプリの音声・通話音声は取得不可 |
| 画面表示 | ◎ 常時前面の小型オーバーレイが作れる | △ アプリを開いている間のみ |
| 向くユースケース | Web会議・オンライン面接・配信 (A/B/C/D) | 対面会話・語学・練習モード (B/E) |
| 開発の速さ | ◎ Pythonで最短 | △ Swift/ネイティブ開発が必要 |

**結論: まずWindowsで作るのが正解。** iPhoneは「対面会話をマイクで拾う」用途に限定されるため、第2弾として練習モード・翻訳モードを移植するのが現実的。

## 4. 技術構成案

### 音声入力
- **Windows**: `sounddevice` / WASAPI loopback — マイク(自分)+スピーカー出力(相手)の2系統を同時キャプチャ
- **iPhone**: `AVAudioEngine` でマイク入力のみ

### 文字起こし(STT) — 選択肢
| 方式 | 遅延 | 日本語精度 | コスト |
|---|---|---|---|
| OpenAI Realtime API / gpt-4o-transcribe | 数百ms | 高 | 従量課金 |
| Deepgram / AssemblyAI ストリーミング | ~300ms | 中〜高 | 従量課金 |
| faster-whisper ローカル実行(GPU推奨) | 0.5〜1秒 | 高 | 無料 |
| iOS標準 SFSpeechRecognizer | 低 | 中 | 無料・オンデバイス |

推奨: MVPはクラウドSTT(実装が簡単)、コスト最適化フェーズでローカルwhisperへ。

### 回答生成(LLM)
- **Claude API(streaming)** を採用。発話が一区切りするたびに直近の会話履歴+事前資料を投げ、回答候補をストリーミング表示
- 速度優先の候補出し = Haiku、精度が必要な商談回答 = Sonnet/Opus と使い分け
- 事前資料はプロンプトキャッシュに載せて低コスト・低遅延化

### UI
- **Windows**: Tauri または Python + WebView の常時最前面オーバーレイ(半透明・クリック透過)。Web会議の画面共有に映らないよう「共有除外ウィンドウ」設定も可能
- **iPhone**: SwiftUI。大きな文字で回答候補をカード表示、タップで詳細展開

## 5. 段階的な開発ロードマップ

1. **フェーズ1 (MVP・Windows)**: マイク+ループバック音声 → クラウドSTT → Claudeで回答候補3件をウィンドウ表示。コンソール/簡易GUIで可
2. **フェーズ2**: 事前資料の読み込み(RAG)、話者分離(自分/相手)、プロンプトキャッシュ、オーバーレイUI
3. **フェーズ3**: 会議後の議事録自動生成、面接練習モード、配信アシスタント連携
4. **フェーズ4 (iPhone)**: 対面会話・語学・練習モードをiOSアプリ化

## 6. コスト目安(1時間の会話あたり)

- クラウドSTT: 約 $0.3〜0.9
- Claude API(Haiku中心+キャッシュ活用): 約 $0.1〜0.5
- 合計 **1時間あたり数十円〜150円程度**。ローカルSTTにすればSTT分はゼロ

## 7. 注意点

- **録音の同意**: 相手側音声を取得・処理する場合、相手の同意を得るのが原則(特に通話・商談)。利用規約上、面接での無断使用を禁じる企業もあるため、練習モード・自分側支援を主用途として設計するのが安全
- **iOSの制約**: 通話や他アプリの音声は技術的に取得不可。iPhone版は「マイクで拾える対面会話」に用途を絞る
- **遅延との戦い**: 「話し終わる前に候補を出し始める」ため、文の途中でも部分テキストをLLMに先行投入する設計(投機的生成)が体感を大きく左右する
